Introduction

This tutorial demonstrates how to exploit a Cross-Site Scripting (XSS) vulnerability in the bWAPP (an intentionally vulnerable web application). We will specifically focus on reflected XSS attacks using both POST and GET methods.

Steps

1. Accessing the XSS Vulnerability Page

  • Open the bWAPP application in a browser.
  • Navigate to the XSS (Reflected) vulnerability page.

2. Setting the Security Level

  • Ensure the security level is set to low to easily exploit the vulnerability.

3. Reflected XSS using POST Method

Step-by-Step Execution:

Open the XSS - Reflected (POST) Page:

URL: http://localhost/bWAPP/xss_post.php

Enter Data and Submit:

  • Enter dev as both the first and last name.
  • Click on the "Go" button.
  • The application reflects the entered data on the page.

Inject XSS Payload:

  • Enter <h1>Hacked</h1> as the first name.
  • Enter dev as the last name.
  • Click on the "Go" button.
  • The application reflects the injected payload, which gets executed in the browser.

4. Reflected XSS using GET Method

Step-by-Step Execution:

Open the XSS - Reflected (GET) Page:

URL: http://localhost/bWAPP/xss_get.php

Enter Data and Submit:

  • Enter dev as both the first and last name.
  • Click on the "Go" button.
  • The application reflects the entered data in the URL and on the page.

Inject XSS Payload:

  • Manually modify the URL to inject the XSS payload:
    • http://localhost/bWAPP/xss_get.php?firstname=<h1>Hacked</h1>&lastname=dev&form=submit
  • Press Enter.
  • The application reflects the injected payload, which gets executed in the browser.
